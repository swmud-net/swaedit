First of all you are probably not the person who should use renumbering. Renumbering is for OLC builders and mud admins only. If you do not know what is OLC than it means you must not use renumbering.

There are 2 types of renumbering. You can see them in the options section. The types:
  1. reliable renumbering renumbers everything reliable, which is everything except mud programs. This is because mud programs are hard to renumber automatically. User action is required after almost every renumber process.
  2. programs renumbering renumbers everything that reliable renumbering does, but additionally renumbers mud programs. After renumbering these programs it displays warnings (if any) and saves them automatically into a file avoiding loss in this way.
Regardless of the option used, you must check all of the mud programs. If you were using reliable renumbering you must renumber all programs manually. If the option used was programs renumbering you must check all of the programs and fix the ones with errors. Here, warnings can be helpful. Warnings are written to the file, which name is composed from area name and placed in the same directory as the area file. The name of this file is the area file name with stripped .xml extension and added _renumberWarnings.txt part. Assuming your area is named coruscant.xml the warnings file name would be: coruscant_renumberWarnings.txt.
